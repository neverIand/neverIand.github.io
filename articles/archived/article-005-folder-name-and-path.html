文件夹路径

低级错误，但我居然没有搜到

在不同的情况下遇到了两次这个问题，所以记录下来。


Vue3使用脚手架初始化项目后运行npm run serve报错Cannot find module '...\node_modules\@vue\cli-service\bin\vue-cli-service.js'。试了各种方法都没有用。最后再次检查控制台输出发现vue cli输出中的项目文件夹名少了一个"^"符号，实际的文件夹名里面是有这个符号的。教训：不要在项目文件夹名里使用"^"这个符号。



前两天装完flutter后使用Android Studio初始化flutter项目失败，但是运行flutter doctor指令后却没有发现任何问题。与此同时，在另一个目录使用flutter create创建的项目却一切正常。同样是在检查控制台输出后，发现项目的路径中包含的"&"被替换成了它的HTML转义字符，于是猜测可能是这个路径名的问题。在另一个位置创建项目后这个猜想被证实，原因就是它。（这门课叫做Mobile & Ubiquitous Computing，我文件夹名是直接复制的）教训：不要在项目文件夹名里使用"&"这个符号。
