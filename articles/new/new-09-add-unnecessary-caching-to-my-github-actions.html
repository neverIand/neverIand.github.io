Anyways, first part of this article is essentially a reminder for myself so that next time when I update the gh actions of this site I can make less silly mistake.

To be clear, I don't think it's necessary to setup caching for this site - it is a static site after all and it doesn't even use npm. It only uses some 'dependencies' in github action for minimizing Javascript and CSS. Particularly, I haven't noticed significant deployment speed different after caching is implemented, so this is only for learning purpose.

Adding caching itself is not that complicated, but it does mean I need to make this project an npm project by running npm init -y and add packages needed for minifying to local dependency. Since the gh action needs to run npm ci, and npm is pointing me with a gun to init the project locally before running that command.

Things to check (because I'm stupid):

  • Whether the action is running on the correct branch
  • Node version - github runners will complain if node version doesn't match
  • make sure to use find command to recursively find js and css files
  • REPLACE the original css and js files after minimizing because I'm not replacing their reference in HTML
  • If the code is not minified, again, check if the workflow is running on the right branch and if the correct branch is being deployed. Then, check the gh-pages branch and see if the minified files are present.
